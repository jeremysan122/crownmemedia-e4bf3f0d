# Security policy

Do not disclose suspected vulnerabilities in a public issue. Use the repository's
[private vulnerability reporting](https://github.com/jeremysan122/crownmemedia-e4bf3f0d/security/advisories/new)
channel and include reproduction steps, affected accounts/data, and an assessment
of impact. Do not access data belonging to other users, disrupt production, or
perform denial-of-service testing.

The team will acknowledge a valid report as quickly as operational coverage
allows, triage severity, preserve evidence, and coordinate a fix and disclosure.
Production secrets exposed in source or logs must be revoked immediately rather
than waiting for a code release.

Only the latest production release is supported. CrownMe currently has no public
bug-bounty or safe-harbor program; written authorization is required for testing
beyond ordinary use of the product.
